Overview

This project basically introduced us to Meshes and how to interact with them.  We started off by evaluating Bezier curves and surfaces using de Casteljau's algorithm, and were able to make smooth curves from simple meshes and points.  Then, we moved on to mesh operations.  Essentially, given a starting mesh of an object, how do we actually manipulate the mesh in order to do what we want!  This involved smoother shading, edge splitting and edge flipping, which enabled us to then implement loop subdivision!  Loop subdivision is probably the coolest thing we have done so far, as it enables us to make a non-smooth rendered object smooth and lifelike!

Part 1

Briefly explain de Casteljau's algorithm and how you implemented it in order to evaluate Bezier curves.

  • De Casteljau’s algorithm is a recursive algorithm for evaluating Bezier curves.  It uses subdivision and linear interpolation as the main mathematical tools to evaluate the curve. At each step, between every two control points, we insert a point using linear interpolation and the formula given in the spec that is t away from the first control point, and 1-t away from the second control point, where t is our input parameter (t lies in the range (0, 1)).  If we have 4 original control points, in the next iteration we will have 3 new control points.  Then this whole algorithm repeats recursively until we only have one control point.  That one control point lies on our final curve.  We repeat the entire above process for a bunch of different values of t, giving us a bunch of different points that lie on the curve!

Part 2

Briefly explain how de Casteljau algorithm extends to Bezier surfaces and how you implemented it in order to evaluate Bezier surfaces.

  • A Bezier surface is defined by a rectangular mesh of control points, almost like multiple Bezier curves stacked horizontally next to each other, and Bezier curves connected to the control points of each original Bezier curve.  To evaluate a Bezier surface using de Casteljau’s algorithm, we apply the algorithm explained in Part 1 in one direction, for example along the rows of the Bezier mesh to obtain a new set of control points.  Once we have these new control points, we apply de Casteljau’s algorithm in the perpendicular direction, for example across the columns to obtain one final point on the surface.  We repeat this entire process for u, v values in the range (0, 1) (u acts as t in the first direction, and v acts as t in the second direction), giving us a 3-D surface.

Part 3

Briefly explain how you implemented the area-weighted vertex normals.

  • We follow a similar process as the example code to loop through all neighboring vertices in half-edge primer; however, instead of traversing every neighboring vertex, we get each face corresponding to the half edge outgoing from our original vertex. We then loop through all the faces by calling h->twin()->next(), which brings us to the next outgoing half-edge from our vertex.
  • Once we have the face, we then compute the area of the face using a helper function that takes the cross product of two triangle edges and dividing them by two.  Once we have this, we then add our normal (obtained using face->normal()) multiplied by our face area to our total sum vector.  Finally, after we have done this for all faces, ending when the half edge we are on is back to the original vertex’s halfedge, we return our normalized unit vector of area-weighted vertex normals.
  • Area Helper Function

int area( Vector3D a, Vector3D b, Vector3D c )

{return cross(b - a, c - a).norm() / 2;}

Part 4

Briefly explain how you implemented the edge flip operation and describe any interesting implementation / debugging tricks you have used.

  • To implement this edge flip operation, the first step in our working code was to gather all the vertices, half edges and faces.  Then, before performing the edge flip, we check to see that the faces are not boundary faces, and if the face is a boundary face, we do nothing and move on.
  • However, if it isn’t then we go through the whole process of reassigning pointers for next, twin, vertex, edge, and face for each half edge that was moved and anything that pointed to it. We chose to move the bc edge to be in the ad location, and then update poniters accordingly (as opposed to the alternative approach of rotating all the vertices/edges). In addition, we must update each vertex to point to one of its outgoing halfedge and each face to point to one of its corresponding halfedges.  This was the most grueling part, as it just took a lot of thinking and typing.

Part 6

Briefly explain how you implemented the loop subdivision and describe any interesting implementation / debugging tricks you have used.

  • To implement loop subdivision, we followed the suggested process in the spec, and reversed the order of the steps (as mentioned in the upsample comments).
  • The first step was to loop through all current vertices and  pre-compute the new positions of all old vertices in the input mesh using float u = (n == 3) ? 3.0/16.0 : 3.0/(8.0*n);
  • Then, we looped through all edges and computed the positions for resulting new vertex that would be created by splitting that edge using the formula e->newPosition = 3.0/8.0 * (a->position + b->position) + 1.0/8.0 * (c->position + d->position. We store these positions in the edge’s newPosition variable.
  • We then loop through all the edges again and split them with splitEdge(). We store the new vertex position calculated in the previous step inside the vertex’s newPosition variable. We only loop through the old edges (by only looping for mesh.nEdges();) so that we don’t split any newly created edges (which otherwise infinite loops, which is an issue we ran into).
  • The second last step is to flip any edges that is a new edge (as marked in split edge) and connects one new vertex to one old vertex (if (e->isNew && (a->isNew != b->isNew))).
  • Finally, we update the positions of the original vertices using the precomputed new positions in the first step!  And that is loop subdivision!
  • Debugging:
  • As mentioned earlier, we had an infinite loop error where we forgot to limit our second edge loop to only loop through the old edges.
  • We had an annoying little bug, where for our code below

for (HalfedgeIter h = v->halfedge(); h != v->halfedge(); h = h->twin()->next())
{
sum += h->twin()->vertex()->position;
}

we never entered the loop, as our end condition was reached with our initial condition for the loop.  This nifty little bug took us a full hour to find, but once we did, we just switched it to a do while loop, which enabled us to do an iteration of a loop and then update h, and then check the condition.
